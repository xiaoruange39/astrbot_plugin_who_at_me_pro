# astrbot_plugin_who_at_me_pro

从 Yunzai 插件 `谁艾特我.js` 移植来的 AstrBot 插件，用来记录群聊中的 `@`，查询谁艾特过自己或指定成员。

## 功能

- 自动记录群聊里的 `@` 消息、图片 URL、发送者、时间。
- 查询 `谁艾特我`、`谁@我`、`谁at我`、`哪个逼艾特我`，也支持 `/谁艾特我`。
- 查询别人：发送 `谁艾特他 @某人`、`谁@她 @某人`、`谁at它 @某人`，也支持前面加 `/`。
- 清除自己的数据：`clear_at` 或 `清除艾特数据`，也支持 `/clear_at`。
- AstrBot 管理员或主人清除全部数据：`clear_all` 或 `清除全部艾特数据`。
- 管理员开关上下文：`开启艾特上下文` / `关闭艾特上下文`，开启后记录被艾特消息前后各 5 条群消息。
- 管理员开关本群被动提醒：`#开启本群艾特提醒` / `#关闭本群艾特提醒`。
- 个人开关被动提醒：`#开启我的艾特提醒` / `#关闭我的艾特提醒`，也支持 `#开启艾特提醒` / `#关闭艾特提醒`。
- 查看提醒状态：`#艾特提醒状态` 或 `#我的艾特提醒状态`。
- 管理员开关提醒截图上下文：`#开启提醒上下文` / `#关闭提醒上下文`。
- 管理员设置提醒截图上下文：`#设置提醒上下文 前条数,后条数`，最大各 5 条。
- WebUI 全局白名单可限制只有指定群 UMO 使用本插件。

## 安装

把整个 `astrbot_plugin_who_at_me_pro` 目录放到 AstrBot 的 `data/plugins/` 下，然后在 WebUI 里重载插件。

## WebUI 配置

插件提供 `_conf_schema.json`，可在 AstrBot WebUI 的插件配置里调整：

- 全局使用：允许使用插件的群 UMO 白名单。
- 记录查询：每人最大记录数、单张图最大消息数、单次查询最多发送页数、查询上下文缓存数、查询结果正序/倒序下拉选择。
- 艾特提醒：默认群开关、群 UMO 名单、默认个人开关、离开判定分钟数、待提醒记录上限、单次提醒最多发送页数、提醒上下文默认配置。
- 图片渲染：是否优先浏览器渲染、JPEG 质量、渲染超时、头图/尾图 URL、临时图片清理时间。
- 提示文案：查询等待提示、回群提醒提示。

WebUI 里的“允许使用插件的群 UMO 白名单”留空时不限制群；填入后只有名单里的 UMO 会记录、查询、清理和触发提醒。被动提醒默认开启本群提醒，也可以在 WebUI 关闭默认群开关，或用 `#开启本群艾特提醒` / `#关闭本群艾特提醒` 单独控制某个群。WebUI 里的“限定开启提醒的群 UMO 名单”只限制提醒功能，留空时不限制提醒群；填入后只有名单里的 UMO 会生成和推送提醒，当前群 UMO 可通过 `#艾特提醒状态` 查看。某个成员超过配置的离开时间没有在群里发言时，如果被别人艾特，插件会把记录放入待提醒队列；该成员下一次在群里发言或使用本插件命令时，会自动推送“你不在的时候有 N 条艾特记录”以及截图。提醒会优先以“文字 + 图片”的单条消息链发送，协议端不支持时自动回退成分开发送。插件刚重载后如果还没有该成员的发言记录，会以插件启动时间作为离开判断基准。

# astrbot_plugin_who_at_me

从 Yunzai 插件 `谁艾特我.js` 移植来的 AstrBot 插件，用来记录群聊中的 `@`，查询谁艾特过自己或指定成员。

## 功能

- 自动记录群聊里的 `@` 消息、图片 URL、发送者、时间。
- 查询 `谁艾特我`、`谁@我`、`谁at我`、`哪个逼艾特我`，也支持 `/谁艾特我`。
- 查询别人：发送 `谁艾特他 @某人`、`谁@她 @某人`、`谁at它 @某人`，也支持前面加 `/`。
- 清除自己的数据：`clear_at` 或 `清除艾特数据`，也支持 `/clear_at`。
- 管理员清除全部数据：`clear_all` 或 `清除全部艾特数据`。
- 管理员开关上下文：`开启艾特上下文` / `关闭艾特上下文`，开启后记录被艾特消息前后各 5 条群消息。
- 管理员开关本群被动提醒：`#开启本群艾特提醒` / `#关闭本群艾特提醒`。
- 个人开关被动提醒：`#开启我的艾特提醒` / `#关闭我的艾特提醒`，也支持 `#开启艾特提醒` / `#关闭艾特提醒`。
- 查看提醒状态：`#艾特提醒状态` 或 `#我的艾特提醒状态`。
- 管理员开关提醒截图上下文：`#开启提醒上下文` / `#关闭提醒上下文`。
- 管理员设置提醒截图上下文：`#设置提醒上下文 前条数,后条数`，最大各 5 条。

## 安装

把整个 `astrbot_plugin_who_at_me` 目录放到 AstrBot 的 `data/plugins/` 下，然后在 WebUI 里重载插件。

## WebUI 配置

插件提供 `_conf_schema.json`，可在 AstrBot WebUI 的插件配置里调整：

- 记录查询：每人最大记录数、单张图最大消息数、查询上下文缓存数。
- 艾特提醒：默认群开关、默认个人开关、离开判定分钟数、待提醒记录上限、提醒上下文默认配置。
- 图片渲染：是否优先浏览器渲染、JPEG 质量、渲染超时、头图/尾图 URL、临时图片清理时间。
- 提示文案：查询等待提示、回群提醒提示。

## 说明

插件使用 AstrBot 的 KV 存储保存记录；查询结果图片优先使用 Playwright/Chromium 浏览器渲染并截图，浏览器不可用时回退到 AstrBot 的 `html_render`，如果渲染或发图失败会发送文本摘要。图片样式按原 Yunzai JS 插件的手机聊天截图结构实现，包括原版头/尾图、顶部群标题、头像、角色标签、蓝色 @ 气泡、底部时间和新消息分隔。插件会优先通过协议端 API 获取机器人群名片/昵称，避免把 QQ 号显示在等待提示里。记录时优先读取当前消息的原始顶层消息段，引用/回复里的 @ 不会算进艾特记录，并会在入库和渲染两处剥离文本里的 @ 展示串，避免历史记录结果图出现两个 @。不需要 Node、Puppeteer 或 Redis。插件用高优先级群消息监听器解析原插件的自然语言命令，因此既支持 `谁艾特我`，也支持 `/谁艾特我`，并会在识别到本插件指令后立即阻止默认 LLM 回复。

`@全体` 会作为全体成员记录保存，查询任意成员时都会合并显示这类记录。

被动提醒默认需要先开启本群提醒。某个成员超过 10 分钟没有在群里发言时，如果被别人艾特，插件会把记录放入待提醒队列；该成员下一次在群里发言时，会自动推送“你不在的时候有 N 条艾特记录”以及截图。
